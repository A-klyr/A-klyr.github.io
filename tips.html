Gunakan Nama Variabel yang Deskriptif

Best Practice

Hindari nama variabel yang ambigu seperti x, data, atau temp. Gunakan nama yang menjelaskan isi atau fungsinya.

# ‚ùå Buruk x = 25 data = "John" # ‚úÖ Bagus student_age = 25 student_name = "John"

Mengapa penting? Kode yang mudah dibaca akan menghemat waktu debugging dan memudahkan kolaborasi dengan developer lain.

Level: Pemula

List Comprehension untuk Kode Lebih Ringkas

Pro Tip

List comprehension membuat kode lebih ringkas dan lebih cepat dibanding loop biasa untuk membuat list.

# ‚ùå Cara biasa squares = [] for i in range(10): squares.append(i ** 2) # ‚úÖ List comprehension squares = [i ** 2 for i in range(10)]

Bonus: Bisa ditambahkan kondisi: [x for x in range(20) if x % 2 == 0]

Level: Menengah

Hati-hati dengan Mutable Default Arguments

Kesalahan Umum

Jangan gunakan list atau dictionary sebagai default argument di function. Ini akan menyebabkan bug yang sulit dilacak!

# ‚ùå Buruk - list akan di-share antar pemanggilan def add_item(item, items=[]): items.append(item) return items # ‚úÖ Bagus def add_item(item, items=None): if items is None: items = [] items.append(item) return items
Level: Pemula

Gunakan enumerate() daripada range(len())

Pro Tip

Saat butuh index dan value sekaligus dalam loop, gunakan enumerate() untuk kode yang lebih pythonic.

# ‚ùå Kurang pythonic fruits = ['apple', 'banana', 'orange'] for i in range(len(fruits)): print(f"{i}: {fruits[i]}") # ‚úÖ Lebih pythonic for i, fruit in enumerate(fruits): print(f"{i}: {fruit}")
Level: Menengah

Gunakan Virtual Environment untuk Setiap Project

Best Practice

Selalu buat virtual environment untuk setiap project Python untuk menghindari konflik dependencies.

# Membuat virtual environment python -m venv venv # Aktivasi (Windows) venv\Scripts\activate # Aktivasi (Mac/Linux) source venv/bin/activate

Tip: Tambahkan venv/ ke file .gitignore agar tidak ter-commit!

Level: Pemula

Gunakan f-strings untuk String Formatting

Pro Tip

F-strings (Python 3.6+) adalah cara tercepat dan paling readable untuk formatting strings.

# ‚ùå Cara lama name = "Alice" age = 25 print("My name is " + name + " and I'm " + str(age)) # ‚úÖ F-strings (recommended) print(f"My name is {name} and I'm {age}") # Bonus: bisa eksekusi expression print(f"Next year I'll be {age + 1}")
Level: Lanjutan

Jangan Lupa Close File atau Gunakan Context Manager

Kesalahan Umum

Selalu gunakan with statement saat bekerja dengan file. File akan otomatis ter-close.

# ‚ùå Rawan lupa close file = open('data.txt', 'r') data = file.read() file.close() # Sering terlupakan! # ‚úÖ Gunakan context manager with open('data.txt', 'r') as file: data = file.read() # File otomatis ter-close di sini
Level: Pemula

Follow PEP 8 Style Guide

Best Practice

PEP 8 adalah panduan style official Python. Mengikutinya membuat kode lebih konsisten dan readable.

# Beberapa aturan penting: # - Indentasi 4 spasi # - Max line length 79 karakter # - 2 blank lines sebelum class/function definition # - snake_case untuk variables dan functions # - PascalCase untuk class names # Install linter untuk otomatis check # pip install flake8 # flake8 your_file.py
Level: Menengah
